# 用户画像：下沉市场首套房决策助手

> **目的**：帮团队在产品设计和功能优先级决策时"心中有具体的人"，而非抽象的用户概念。

---

## 画像A：小王 —— "纯小白，完全不知道从哪开始"

### 基本信息

| 维度 | 描述 |
|------|------|
| **姓名** | 小王 |
| **年龄** | 27岁 |
| **城市** | 湖南省浏阳市 |
| **职业** | 本地电商公司运营 |
| **月收入** | 6,500元（到手约5,500） |
| **家庭月收入** | 约10,000元（父母同住，有少量补贴） |
| **当前储蓄** | 15万元 |
| **婚恋状态** | 有女友，计划2年内结婚 |
| **租房情况** | 目前租房，月租1,200元 |
| **购房经历** | 零，从未看过房 |

### 行为模式

- 下班后刷抖音/小红书看"买房攻略"，但感觉信息良莠不齐
- 不知道首付到底要准备多少钱——网上说的20%和实际的"落地成本"差距很大
- 去过两次楼盘，销售说的月供和他自己算的对不上
- 最大的困惑：**"我到底买得起多少钱的房子？"**

### 核心诉求

1. **"买得起"计算**：不是简单的总价×首付比例，而是包含税费、中介费的真实上车门槛
2. **月供安全感**：想看到"月供占收入比例"是否在安全范围内
3. **避免被坑**：不想被中介/销售忽悠买超过承受能力的房子
4. **简单直接**：不需要复杂的数据分析功能，只需要明确的答案

### 决策权重（用户自述）

> "价格最重要吧...然后是面积够用就行，新房当然更好但也能接受二手，通勤不太在意因为浏阳不大。"

| 维度 | 重要性（1-5） | 说明 |
|------|:---:|------|
| 总价 | ★★★★★ | 预算卡得很死 |
| 月供 | ★★★★★ | 不能超过收入一半 |
| 面积 | ★★★☆☆ | 80-100㎡就行 |
| 房龄 | ★★★☆☆ | 不要超过20年 |
| 地段 | ★★☆☆☆ | 浏阳市区交通差别不大 |

### 使用场景

```
周五晚上 9:30，小王躺床上刷手机
→ 打开"首套房决策助手"
→ 输入：月收入6500，储蓄15万，想买80-100㎡
→ 系统告诉他："你真正买得起的总价上限是62万，推荐月供范围1800-2600元"
→ 他看到浏阳市区均价55万，心里有底了
→ 系统推荐了20套匹配房源，Top3中有一套58万的3室2厅
→ 他下载了PDF报告，决定周末去看这套
```

## 画像B：小张夫妇 —— "看过好多套，越看越纠结"

### 基本信息

| 维度 | 描述 |
|------|------|
| **姓名** | 小张（夫，31岁）+ 小李（妻，29岁） |
| **城市** | 广东省湛江市 |
| **职业** | 夫：中学教师；妻：银行柜员 |
| **家庭月收入** | 15,000元（到手约12,500） |
| **当前储蓄** | 30万元 |
| **子女情况** | 备孕中，计划2年内要孩子 |
| **租房情况** | 租房，月租2,000元 |
| **购房经历** | 已看房3个月，实地看过20+套 |

### 行为模式

- 周末经常一起去看房，回来后在Excel里记录对比
- 在贝壳/安居客上收藏了50+套房源
- 已经在A小区（新房，120㎡，总价95万）和B小区（二手房，100㎡，总价72万）之间纠结了一个月
- 丈夫倾向A（新房省心），妻子倾向B（总价低、月供压力小）
- 最大的困惑：**"到底选哪个？谁能给个客观的判断？"**

### 核心诉求

1. **客观对比**：不想只听销售/中介说，想要数据说话
2. **长期视角**：未来有孩子后需要更大空间，现在的选择要考虑5-10年
3. **成本全貌**：不只是总价，还有装修（新房也是一笔钱）、物业费、未来的置换成本
4. **决策信心**：需要"有人告诉我选哪个是对的"——或者至少有明确的决策框架

### 决策权重（夫妻协商后）

> "面积最重要——以后有孩子至少要3房；然后是月供不能太紧张；通勤也要考虑因为要接送孩子。"

| 维度 | 重要性（1-5） | 说明 |
|------|:---:|------|
| 面积 | ★★★★★ | 至少100㎡，3房是底线 |
| 月供/总价 | ★★★★☆ | 月供不超过家庭收入40% |
| 户型/朝向 | ★★★★☆ | 南北通透优先 |
| 通勤 | ★★★☆☆ | 离学校近加分 |
| 房龄 | ★★★☆☆ | 二手10年内可接受 |

### 使用场景

```
周日下午，小张夫妇坐在电脑前
→ 打开"首套房决策助手"
→ 输入家庭月收入15000，储蓄30万，偏好3房100㎡以上
→ 系统推荐列表里同时出现了A小区和B小区的房源
→ 使用"对比"功能，并排比较两个小区：
   - A小区：总价95万，月供4,200元（占收入28%），新房0年房龄
   - B小区：总价72万，月供3,100元（占收入21%），8年房龄
→ 系统提示："B小区综合得分更高（87 vs 72），主要优势在总价压力和月供负担。但A的新房优势在装修成本预估中需额外增加8-10万。"
→ 使用敏感度分析："如果把面积权重调到最高，结果有什么变化？"
→ 小张夫妇达成共识：选B小区，省下的钱可以作为装修和育儿基金
→ 导出PDF报告，准备下周去谈B小区
```

## 画像C：老刘 —— "改善型需求，但不清楚划不划算"

### 基本信息

| 维度 | 描述 |
|------|------|
| **姓名** | 老刘 |
| **年龄** | 42岁 |
| **城市** | 山东省昌邑市（县级市） |
| **职业** | 本地建材店老板 |
| **月收入** | 8,000元（个体经营，波动较大） |
| **家庭月收入** | 约12,000元 |
| **当前储蓄** | 25万元 |
| **子女情况** | 一子（15岁，即将上高中） |
| **当前住房** | 自有一套80㎡两房（已还清贷款），房龄18年 |
| **购房经历** | 15年前买过一套，但当时是"熟人介绍随便买的" |

### 行为模式

- 现在住的房子太小（孩子大了，两房不够用）、太老（漏水/隔音差）
- 想换一套大的，但不确定"卖了旧的买新的"在财务上是否划算
- 对贷款比较抗拒（年纪大了，不想背太多债）
- 关注的是"改善"而非"刚需"，决策没有那么紧迫
- 最大的困惑：**"换房这件事，到底是亏了还是赚了？"**

### 核心诉求

1. **卖旧买新的价差分析**：当前房子值多少钱？卖了能拿出多少首付？
2. **持有成本对比**：新房子10年下来，利息+物业+折旧一共要花多少钱？
3. **租房vs买房**：偶尔也在想，是不是干脆租一套大的算了？
4. **贷款不焦虑**：想看到不同贷款年限下的月供差异（15年 vs 20年 vs 25年）

### 决策权重

> "面积一定要大，至少120㎡。房龄不能太老，不想再住老房子了。总价当然越低越好，但也不想太远。"

| 维度 | 重要性（1-5） | 说明 |
|------|:---:|------|
| 面积 | ★★★★★ | 120-150㎡，三房以上 |
| 房龄 | ★★★★☆ | 10年以内 |
| 总价 | ★★★★☆ | 80万以内 |
| 贷款年限 | ★★★☆☆ | 倾向于15-20年 |
| 地段 | ★★☆☆☆ | 城区内即可 |

### 使用场景

```
周一下午，老刘在店里空闲时打开电脑
→ 打开"首套房决策助手"
→ 选择"改善型购房"模式
→ 输入：当前房产估值约35万（可卖掉），储蓄25万，家庭月收入12000
→ 系统计算出：
   - 卖旧房后首付可达60万（35+25）
   - 买一套75万的120㎡，贷款15万，月供仅900元（15年）
   - 或者买一套90万的140㎡新房，贷款30万，月供1,900元（15年）
→ 使用"持有成本对比"：
   - 继续住旧房（不换）：10年维修+翻新预估6-8万
   - 换75万次新房：10年总计花费约95万（含利息）
   - 换90万新房：10年总计花费约115万（含利息+装修）
→ 老刘觉得75万方案最合理，导出PDF，回去和老婆商量
```

## 反用户画像（明确排除）

这些用户不是我们产品的目标——没必要为他们设计功能：

- **投资客（多套房持有者）**：核心诉求是租金回报率、升值空间预测——需要完全不同的数据和分析模型
- **北上广深购房者**：贷款政策（公积金上限、首付比例、限购政策）差异巨大，且我们的数据在一线城市覆盖不足
- **已经有明确目标的搜索型用户**：他们需要的是"搜到那套房子"，而不是"被推荐"——直接用贝壳/链家更高效
