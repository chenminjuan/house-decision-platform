## Phase 2: 深度化 —— "好用"（第5-8周）

**核心目标**：让推荐更精准、用户更有掌控感。

### 功能开发

#### 持有成本分析（新增）
- [ ] 10年持有成本模拟器
  - 输入：总价、利率、贷款年限、物业费估算
  - 输出：10年总支出（利息+物业费+装修折旧）、年均持有成本
- [ ] 租房 vs 买房对比
  - 输入：当前租金、预期房价涨幅、预期租金涨幅
  - 输出：10年累计支出对比

#### 敏感度分析（新增）
- [ ] 权重变动对Top10推荐的影响可视化
- [ ] "如果预算增加10万，推荐结果有什么变化？"
- [ ] "如果面积权重翻倍，Top10会变吗？"

#### 评分算法升级（改进）
- [ ] 研究 AHP（层次分析法）替代线性权重
- [ ] 研究 TOPSIS 替代线性加权
- [ ] A/B 测试框架：两套算法的对比

#### 用户反馈机制（新增）
- [ ] 推荐房源"有用/无用"投票按钮
- [ ] 报告末尾的 NPS 评分
- [ ] 反馈数据本地存储

#### 数据更新管道（新增）
- [ ] 数据更新脚本（定期从房天下获取新数据）
- [ ] 数据版本管理（记录每次更新日期）
- [ ] 数据质量指标面板

### Phase 2 交付物
- 持有成本分析功能
- 敏感度分析功能
- A/B 测试框架
- 用户反馈数据收集

## 未来探索（Beyond Phase 3）

这些不在当前路线图中，但值得在产品演进中考虑：

- **社区化**：用户分享自己的购房决策过程和结果
- **数据众包**：用户贡献本地房价数据换取分析权益
- **AI 咨询**：接入 LLM 做对话式购房咨询（"我的情况是XXX，你觉得我该买哪？"）
- **政策解读**：各城市购房政策的自动化解读和影响分析
- **小程序版本**：微信小程序原生体验

## 风险与依赖

| 风险 | 影响阶段 | 缓解计划 |
|------|:---:|------|
| 数据时效性不足 | Phase 1-3 | 透明标注数据日期；Phase 2 建更新管道 |
| Streamlit 表现力有限 | Phase 3 UI | 可接受；如果严重影响体验，考虑迁移到 React+FastAPI |
| 用户获取困难 | Phase 2-3 | 内容营销（小红书/知乎）+ 开源社区曝光 |
| 时间不足 | Phase 2-3 | MVP 是底线，Phase 2/3 可按需裁剪 |
