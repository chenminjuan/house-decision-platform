# 竞品分析：房地产购房决策工具

> **目的**：系统对比市场上的主要竞品，明确差异化定位，为产品设计决策提供依据。

---

## 1. 竞品全景图

按"用户决策深度"和"三四线城市覆盖"两个维度划分竞品矩阵：

```
                    三四线城市覆盖强
                          │
          ★ 房天下        │    ★ 本产品（目标位）
          （信息展示）      │    （决策辅助）
                          │
  浅 ─────────────────────┼──────────────────────→ 深
  （找信息）              │                    （做决策）
                          │
          ★ 安居客        │    ★ 贝壳找房

```

## 2. 逐一竞品分析

### 2.1 贝壳找房 (ke.com)

| 维度 | 描述 |
|------|------|
| **一句话定位** | 技术驱动的品质居住服务平台 |
| **核心模式** | 线上信息撮合 + 线下经纪人服务，ACN 经纪人合作网络 |
| **三四线覆盖** | 中等 —— 主要覆盖地级市，县城覆盖不足 |
| **数据能力** | 强 —— 有"楼盘字典"（户型图/VR/真实成交价） |
| **决策辅助** | 弱 —— 提供房贷计算器（首付/月供），但仅作为工具，不与房源推荐深度集成 |
| **个性化** | 弱 —— 只有筛选（条件查询），无主动推荐和评分排序 |

**贝壳的房贷计算器 vs 我们的购买力评估**：

| 维度 | 贝壳房贷计算器 | 我们的购买力评估引擎 |
|------|:---:|:---:|
| 计算首付 | ✅ | ✅ |
| 计算月供 | ✅（等额本息） | ✅（等额本息+等额本金） |
| 计算税费 | ❌ 需要自己查 | ✅ 自动估算契税/个税/增值税 |
| 计算中介费 | ❌ | ✅ 自动估算 |
| 关联房源推荐 | ❌ 独立工具 | ✅ 直接筛选"买得起"的房源 |
| "买得起上限" | ❌ | ✅ 反向计算真实购买力边界 |
| 收入安全线 | ❌ | ✅ 银行审批红线 + 推荐舒适线 |

**启示**：贝壳的核心价值在"信息真实"和"交易安全"，不在"决策辅助"。这是一个明确的差异化空间。

---

### 2.2 安居客 (anjuke.com)

| 维度 | 描述 |
|------|------|
| **一句话定位** | 房产信息服务平台（流量+广告模式） |
| **核心模式** | 信息聚合 + 经纪人付费推广 + 广告收入 |
| **三四线覆盖** | 中等 —— 覆盖城市广但数据质量参差不齐 |
| **数据能力** | 中 —— 数据来源以经纪人上传为主，真实性和时效性问题突出 |
| **决策辅助** | 极弱 —— 基本只有筛选和排序，无任何分析功能 |
| **个性化** | 无 |

**启示**：安居客本质是广告平台，不是用户决策工具。其"虚假房源"问题长期被用户诟病，反而是我们建立信任的机会。

---

### 2.3 链家 (lianjia.com)

| 维度 | 描述 |
|------|------|
| **一句话定位** | 真房源、真服务（贝壳旗下的直营经纪品牌） |
| **核心模式** | 直营经纪人 + 闭环交易 |
| **三四线覆盖** | 弱 —— 主要聚焦一二线城市 |
| **数据能力** | 极强 —— 成交数据最真实，但三四线城市数据量有限 |
| **决策辅助** | 弱 —— 有"小区评分"但维度单一（综合评分一个数字），无个性化推荐 |
| **个性化** | 无 |

**启示**：链家在一线城市是标杆，但在三四线城市覆盖严重不足。这给了我们在下沉市场做差异化切入的空间。

---

### 2.4 房天下 (fang.com)

| 维度 | 描述 |
|------|------|
| **一句话定位** | 中国最早的房地产家居网络平台 |
| **核心模式** | 信息展示 + 新房导流 + 广告 |
| **三四线覆盖** | 强 —— 覆盖城市最多，是我们的数据来源 |
| **数据能力** | 中 —— 挂牌数据量大，但数据清洗和分析能力弱 |
| **决策辅助** | 无 —— 纯粹的信息展示 |
| **个性化** | 无 |

**启示**：房天下是"数据源"而非"决策工具"。它提供了海量原始数据，但没有做任何分析加工。这正是我们的切入点——同样的数据，做出完全不同的价值。

## 3. 间接竞品

### 3.1 小红书/抖音 房产内容

| 维度 | 描述 |
|------|------|
| **模式** | KOL/经纪人发布"买房攻略""避坑指南"等内容 |
| **优势** | 用户信任度高（人格化推荐）、决策影响力强 |
| **劣势** | 非结构化、无法个性化、信息碎片化、质量良莠不齐 |
| **对我们的启示** | 决策报告可以借鉴"叙事化"的表达方式，不只是冷冰冰的数据 |

### 3.2 Excel 自制比房表

| 维度 | 描述 |
|------|------|
| **模式** | 用户自己用 Excel 记录看过的房源并手动对比 |
| **优势** | 完全个性化、用户完全掌控 |
| **劣势** | 费时费力、数据不完整、缺乏分析能力 |
| **对我们的启示** | 对比功能要做成"自动化 Excel"——用户掌控感 + 系统分析能力 |

### 3.3 银行贷款计算器（各银行App/网站）

| 维度 | 描述 |
|------|------|
| **模式** | 独立的房贷计算工具 |
| **优势** | 计算准确，与银行实际利率对接 |
| **劣势** | 与房源数据完全割裂；只算月供不给建议 |
| **对我们的启示** | 利率数据要对接真实LPR；计算结果要与房源直接关联 |

## 4. 差异化定位宣言

### 我们的位置

```
竞品做的事：                        我们做的事：

贝壳 —— "这些房源是真的"            "这些房源中，哪些适合你"
链家 —— "我们的经纪人帮你买"          "数据帮你做判断，不依赖经纪人"
安居客 —— "这里有好多房源广告"        "过滤噪音，只看匹配的20套"
房天下 —— "全国房源都在这里"          "10万套房源中选出你的Top20"
小红书 —— "过来人告诉你买房经验"      "用算法和计算代替经验主义"
```

### 一句话差异化

> **"贝壳帮你找房，我们帮你做决定。"**

### 三个核心差异

| # | 差异点 | 竞品现状 | 我们的做法 |
|---|--------|---------|-----------|
| 1 | **从总价到落地价** | 都只展示总价 | 首付+税费+中介费+装修=真实门槛 |
| 2 | **从筛选到推荐** | 都是条件筛选+默认排序 | 基于用户画像的多维度加权评分 |
| 3 | **从浏览到报告** | 都是看房源列表 | 可导出的结构化决策报告 |

## 5. 竞争优势的可持续性

### 为什么大厂不会做这件事？

1. **商业模式冲突**：贝壳/链家的收入来自交易佣金——推荐"最适合用户的"和推荐"佣金最高的"存在利益冲突。作为独立决策工具，我们没有这个冲突。
2. **市场优先级**：大厂的首要战场在一二线城市。三四线城市的精细化决策工具 ROI 不支撑大厂的资源投入。
3. **数据壁垒**：我们不需要颠覆贝壳的交易闭环——只需要在"信息→决策"这个环节做到极致。

### 我们的护城河

1. **评分算法的透明度**：公开评分逻辑，用户可以理解"为什么推荐这套"——建立信任
2. **购买力评估的精准度**：持续优化落地成本计算的准确性，成为"买房前的第一步"
3. **下沉市场的深度理解**：画像文档、场景分析沉淀为领域知识

| 竞品 | 一句话 | 三四线 | 决策辅助 | 个性化 | 威胁等级 |
|------|--------|:---:|:---:|:---:|:---:|
| 贝壳找房 | 交易平台 | ★★★ | ★★ | ★ | 🟡 中 |
| 安居客 | 广告平台 | ★★★ | ★ | ☆ | 🟢 低 |
| 链家 | 经纪人模式 | ★★ | ★★ | ★ | 🟢 低 |
| 房天下 | 信息聚合 | ★★★★★ | ☆ | ☆ | 🟢 低 |
| 小红书/抖音 | 内容社区 | ★★★★ | ★★ | ★★★ | 🟡 中 |
| **本产品** | **决策辅助** | ★★★★ | ★★★★★ | ★★★★★ | - |

**结论**：在"三四线城市 + 购房决策辅助"这个交叉点上，目前没有直接竞品。这是一个蓝海缝隙。
